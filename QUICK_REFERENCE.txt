# Quick Reference - StockInvoice Pro Authentication Setup

## 🚀 5-Minute Setup

### 1. Create Supabase Project
- Go to https://app.supabase.com
- Create new project named `stock-invoice`
- Wait for initialization (2-3 min)

### 2. Get Credentials
- Settings → API
- Copy Project URL and anon key
- Save them safely

### 3. Setup Database
- SQL Editor → New Query
- Copy ALL SQL from SUPABASE_SETUP.md
- Run each block (users, activation_keys, stocks, invoices)
- Enable RLS policies from that file too

### 4. Update Config
Open `supabase-config.js`:
```javascript
const SUPABASE_URL = 'https://YOUR_ID.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR_KEY';
```

### 5. Create First Admin
1. Open http://localhost/stock-invoice
2. Sign Up with activation key (generate in Supabase SQL)
3. In Supabase: Set role='admin' and status='approved'
4. Go to http://localhost/stock-invoice/admin.html

## 👤 User Flow

```
1. User visits http://localhost/stock-invoice
   ↓
2. User clicks "Sign Up"
   ↓
3. User enters email, password, activation key
   ↓
4. System verifies key is valid
   ↓
5. User account created with status="pending"
   ↓
6. Admin reviews in "Pending Approvals"
   ↓
7. Admin clicks "Approve"
   ↓
8. User receives approval email
   ↓
9. User can now login
   ↓
10. User sees their personal stock and invoices
```

## 🗄️ Database Tables

1. **user_profiles** - User accounts & status
2. **activation_keys** - One-time signup codes
3. **user_stocks** - Each user's products
4. **user_invoices** - Each user's invoices

Each user can only see their own data (RLS enabled)

---

## 🔓 Security Features

✓ Passwords hashed by Supabase  
✓ Row-level security (users see only their data)  
✓ Activation keys expire in 30 days  
✓ Admin approval required to login  
✓ Account suspension available  

---

## ❌ Common Mistakes

1. **Not running ALL SQL blocks** → Tables won't exist
2. **Wrong credentials in config** → "Connection failed"
3. **User status not set to 'approved'** → "Awaiting approval"
4. **RLS not enabled** → Users can see all data
5. **Activation key typo** → "Invalid key"

---

## 📞 If Something Goes Wrong

1. Check `SUPABASE_SETUP.md` for SQL troubleshooting
2. Review Supabase Logs tab for errors
3. Verify credentials in `supabase-config.js`
4. Check browser console (F12) for JavaScript errors
5. Ensure RLS policies are created
